# Pourquoi j’ai utilisé Iris avec Scikit-learn

## Contexte

Au départ, je voulais télécharger un dataset depuis Kaggle pour alimenter mon pipeline.
Cependant, je n’ai pas pu récupérer les données correctement depuis Kaggle (problème d’accès/configuration API dans mon environnement).

## Solution adoptée

Pour avancer sans bloquer le projet, j’ai utilisé le dataset **Iris** fourni directement par **Scikit-learn**.
Ce choix permet de continuer le développement et les tests de la pipeline sans dépendre d’un téléchargement externe.

## Pourquoi Iris est un bon choix

- Dataset déjà intégré à `scikit-learn` (pas de téléchargement manuel)
- Léger et rapide à charger
- Format propre pour faire des tests ML
- Très utilisé pour valider une première chaîne de traitement

## Exemple utilisé dans le projet

Dans `dataset/load.py`, on charge Iris avec :

```python
from sklearn.datasets import load_iris
import pandas as pd

data = load_iris()
df = pd.DataFrame(data.data, columns=data.feature_names)
df['target'] = data.target
print(df.head())
```

## Conclusion

Je n’ai pas pu télécharger le dataset depuis Kaggle pour le moment, donc j’ai utilisé **Scikit-learn + Iris** comme alternative fiable.
Cela me permet de continuer la mise en place du pipeline CI/CD pendant que je corrige l’intégration Kaggle.
