# Déploiement Docker Hub & GitHub Actions

Ce guide reprend les étapes réalisées dans le terminal pour déployer le projet sur Docker Hub.

## 1) Préparer les identifiants Docker Hub

- Créer un token Docker Hub avec permission **Read & Write**.
- Ajouter dans GitHub (`Settings` → `Secrets and variables` → `Actions`) :
  - `DOCKERHUB_USERNAME`
  - `DOCKERHUB_TOKEN`

## 2) Vérifier le workflow de déploiement

Le workflow utilise `backend/dockerfile` et `frontend/dockerfile` et pousse vers :
- `asma1315/backend:latest`
- `asma1315/frontend:latest`

Commit/push du workflow (si modifié) :

```zsh
git add .github/workflows/deploy.yml
git commit -m "ci: deploy backend/frontend to Docker Hub"
git push origin main
```

## 3) Relancer le workflow GitHub Actions

- Ouvrir GitHub → `Actions` → `Simple Docker Deploy`
- Cliquer `Run workflow`
- Choisir la branche `main`

Le workflow doit passer en vert sur :
- `Validate Docker Hub secrets`
- `Login Docker Hub`
- `Build and push backend`
- `Build and push frontend`

## 4) Vérifier en local avec Docker

Connexion Docker Hub :

```zsh
docker logout
printf '%s' 'NOUVEAU_DOCKERHUB_TOKEN' | docker login -u asma1315 --password-stdin
```

Vérifier les images publiées :

```zsh
docker pull asma1315/backend:latest
docker pull asma1315/frontend:latest
```

## 5) Dépannage rapide

- `authentication required`: token invalide ou expiré → régénérer token + mettre à jour `DOCKERHUB_TOKEN`.
- `not found`: image pas encore poussée → vérifier run Actions et tags Docker Hub.
- Secrets vides dans Actions: vérifier noms exacts des secrets (`DOCKERHUB_USERNAME`, `DOCKERHUB_TOKEN`).

## 6) Sécurité

- Ne jamais commit `.env`.
- Révoquer immédiatement tout token exposé.
- Garder les credentials uniquement dans GitHub Secrets.
