# Guide de déploiement

Ce document rassemble les étapes manuelles de déploiement utilisées sur le projet :

- publication des images sur Docker Hub,
- automatisation via GitHub Actions,
- mise en ligne sur Azure App Service (conteneurs Linux).

## 1) Déploiement sur Docker Hub

### 1.1 Préparer les credentials

1. Créer un token Docker Hub avec permission **Read & Write**.
2. Dans GitHub : `Settings` → `Secrets and variables` → `Actions`.
3. Ajouter les repository secrets :
   - `DOCKERHUB_USERNAME`
   - `DOCKERHUB_TOKEN`

### 1.2 Push des images (manuel)

```zsh
docker logout
printf '%s' 'NOUVEAU_DOCKERHUB_TOKEN' | docker login -u asma1315 --password-stdin

docker build -f backend/dockerfile -t asma1315/backend:latest .
docker push asma1315/backend:latest

docker build -f frontend/dockerfile -t asma1315/frontend:latest .
docker push asma1315/frontend:latest
```

### 1.3 Vérifier les images publiées

```zsh
docker pull asma1315/backend:latest
docker pull asma1315/frontend:latest
```

## 2) Déploiement automatique avec GitHub Actions

Le workflow `Simple Docker Deploy` :

- valide les secrets Docker Hub,
- se connecte au registry,
- build et push :
  - `asma1315/backend:latest`
  - `asma1315/frontend:latest`

### 2.1 Relancer manuellement le workflow

1. Ouvrir GitHub → `Actions`.
2. Choisir `Simple Docker Deploy`.
3. Cliquer `Run workflow`.
4. Choisir la branche `main`.

### 2.2 Vérifier le succès

Les étapes suivantes doivent être vertes :

- `Validate Docker Hub secrets`
- `Login Docker Hub`
- `Build and push backend`
- `Build and push frontend`

## 3) Déploiement sur Azure App Service (manuel)

### 3.1 Backend

Dans l’App Service backend :

1. Aller dans `Deployment Center`.
2. Source : `Container Registry`.
3. Registry : `Docker Hub`.
4. Image : `asma1315/backend`.
5. Tag : `latest`.
6. Sauvegarder.

Puis dans `Configuration` → `Application settings`, ajouter :

- `WEBSITES_PORT=8000`

Redémarrer l’application backend.

### 3.2 Frontend

Dans l’App Service frontend :

1. Aller dans `Deployment Center`.
2. Source : `Container Registry`.
3. Registry : `Docker Hub`.
4. Image : `asma1315/frontend`.
5. Tag : `latest`.
6. Sauvegarder.

Puis dans `Configuration` → `Application settings`, ajouter :

- `WEBSITES_PORT=8501`
- `BACKEND_API_URL=https://<URL_BACKEND>/predict`

Redémarrer l’application frontend.

## 4) Dépannage rapide

- `authentication required`: token invalide/expiré.
  - Regénérer le token Docker Hub.
  - Mettre à jour `DOCKERHUB_TOKEN` dans GitHub Secrets.
- `not found` lors du `docker pull`:
  - vérifier que l’image a bien été push,
  - vérifier le nom exact du repository/tag.
- Secrets vides dans Actions:
  - vérifier les noms exacts `DOCKERHUB_USERNAME` et `DOCKERHUB_TOKEN`.

## 5) Bonnes pratiques sécurité

- Ne jamais commit `.env`.
- Révoquer immédiatement tout token exposé.
- Stocker les secrets uniquement dans GitHub Secrets.
