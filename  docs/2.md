# Guide d'exécution complet (Backend + Frontend + Docker)

## Objectif

Cette page regroupe les étapes essentielles pour lancer le projet `Pipeline-CI-CD` proprement, en local et avec Docker.

## Architecture rapide

- `backend/app/main.py` : API FastAPI pour la prédiction Iris
- `backend/ml/train.py` : entraînement + sauvegarde du modèle
- `backend/model/model.pkl` : modèle utilisé par l'API
- `frontend/app.py` : interface Streamlit
- `docker_compose.yml` : orchestration backend + frontend

## Prérequis

- Docker Desktop installé et démarré
- Port `8000` libre pour le backend
- Port `8501` libre pour le frontend

## Lancer avec Docker Compose (recommandé)

Depuis la racine du projet :

```bash
./run.sh
```

Ou manuellement :

```bash
docker compose -f docker_compose.yml up --build
```

Accès :

- Frontend : `http://127.0.0.1:8501`
- Swagger API : `http://127.0.0.1:8000/docs`

Pour arrêter :

```bash
./stop.sh
```

## Entraîner le modèle (si besoin)

Le backend attend un modèle à cet emplacement : `backend/model/model.pkl`.

Pour le régénérer :

```bash
cd backend/ml
python train.py
```

## Format d'appel API

Endpoint : `POST /predict`

Payload :

```json
{
	"sepal_length": 5.1,
	"sepal_width": 3.5,
	"petal_length": 1.4,
	"petal_width": 0.2
}
```

Réponse :

```json
{
	"prediction": 0,
	"species_name": "setosa"
}
```

## Troubleshooting

### 1) `port is already allocated` sur `8000`

Un ancien conteneur utilise déjà le port.

```bash
docker rm -f pipeline-backend pipeline-frontend 2>/dev/null || true
docker compose -f docker_compose.yml down --remove-orphans
docker compose -f docker_compose.yml up --build
```

### 2) Frontend affiche `Connection refused`

Vérifie que le backend est bien démarré et que l'URL API est correcte.

En mode Docker Compose, l'URL interne est déjà configurée en :

`http://backend:8000/predict`

### 3) Vérifier l'état des services

```bash
docker compose -f docker_compose.yml ps
docker compose -f docker_compose.yml logs -f
```

## Bonnes pratiques

- Utiliser `./run.sh` et `./stop.sh` au lieu de lancer les conteneurs un par un.
- Garder le modèle à jour en relançant `train.py` après une modification ML.
- Vérifier `http://127.0.0.1:8000/docs` avant de tester le frontend.
